#最常用CSS选择器解析

*

对页面的每个元素都起作用,通常用来设置所有元素的margin和padding为0

  • { margin: 0; padding: 0; }

也可以在子选择器中使用

#container * { border: 1px solid black; }

除非必要，我不建议在页面中过的的使用星状选择符，因为他的作用域太大，相当耗浏览器资源.

#x

id选择器,具有高优先级(优先级基数为100，远高于class的10),缺点也很明显优先级过高，重用性差

.x

类选择器

x y

后代选择器.选取x元素下的子孙元素y.
注意:它匹配的是所有的后代子孙元素y

x

标签选择器.优先级仅比*高

x:visited

伪类选择器.设置超链接a在其链接地址已被访问过时的样式

x:link

伪类选择器.设置超链接a在未被访问前的样式

x:hover

伪类选择器.设置元素在其鼠标悬停时的样式

x:active

伪类选择器.设置元素在被用户激活（在鼠标点击与释放之间发生的事件）时的样式。

注意，a:hover 必须位于 a:link 和 a:visited 之后，a:active 必须位于 a:hover 之后
可靠的顺序是：l(link)ov(visited)e h(hover)a(active)te, 即用喜欢(love)和讨厌(hate)两个词来概括

x+y

相邻选择器.匹配x后面的第一个y.(只匹配第一个元素)

x>y

子选择器.与后代选择器X Y不同的是，子选择器只对X下的直接子级Y起作用(不是子孙,是子)
IE6不支持

x~y

相邻选择器.与前面提到的X+Y不同的是，X~Y匹配与X相同级别的所有Y元素，而X+Y只匹配第一个

x[title]

a[title] {color: green;}

属性选择器。比如上述代码匹配的是带有title属性的链接元素。

x[title=”foo”]

a[href=”xxx”] {color: #1f6053;}

属性选择器.上面的代码匹配所有拥有href属性，且href为xxx的所有链接。

x[title*=”abc.net”]

属性选择器.选择x元素中的title属性且值包含abc.net的元素

x[href^=”http”]

属性选择器.选择属性为herf且属性值为http开头的x元素

x[foo~=”bar”]

属性选择器.选择属性为foo且属性值包含bar的x元素,该选择器应用在属性值有空格的地方

此例，将会命中1, 3两个div，因为匹配到了class属性，且属性值中有一个值为a

x:checked

checked伪类用来匹配处于选定状态的界面元素，如radio、checkbox

x:after

清除浮动:

x:nth-child(n)

伪类.选择第n个元素(从1开始算起)的样式

x:nth-last-child(n)

伪类.与X:nth-child(n)功能类似，不同的是它从一个序列的最后一个元素开始算起

x:nth-of-type(n)

与X:nth-child(n)功能类似，不同的是它匹配的不是某个序列元素，而是元素类型。例如上面的代码设置页面中出现的第三个无序列表ul的边框。

x:nth-last-of-type(n)

与X:nth-of-type(n)功能类似，不同的是它从元素最后一次出现开始算起。上面例子中设定倒数第三个无序列表的边框

x:first-child

匹配父元素的第一个子元素x

x:last-child

匹配父元素的最后一个子元素x
